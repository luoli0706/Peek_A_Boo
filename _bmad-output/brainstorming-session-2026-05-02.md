**可用技巧：** 全部 26 种游戏头脑风暴技巧

### 调整后多视角洞察

**躲藏者玩家视角：** "我要在倒计时结束前找到一个完美的位置。听到寻找者的脚步声越来越近，我的心跳加速。他走过去了！但他会不会回头？"

**寻找者玩家视角：** "这个房间我知道有3个可能的藏点。那个角落里露出了一只鞋子吗？不是...等等，那个窗帘是不是在动？抓住你了！"

**设计师视角：** PVP的本质是人与人之间的心理博弈。设计最小化——只需要规则和时间压力，剩下的由玩家之间的互动自然生成。关键不是设计内容，而是设计"产生有趣互动的空间"。

**开发者视角（C#基础薄弱）：** PVP意味着网络同步是最大技术挑战。MVP可以先用局域网联机方案（减少网络复杂度）。FPS模板提供第一人称视角和移动，主要新增：多人联机框架、回合状态机、简单的计分板。

### 操作映射（极简按键方案）

| 按键 | 功能 | 适用范围 |
|------|------|----------|
| W/A/S/D | 前后左右移动 | 全部玩家 |
| Space | 跳跃 | 全部玩家 |
| Ctrl（按住） | 蹲伏 | 全部玩家（若动画支持） |
| 鼠标移动 | 视角旋转 | 全部玩家 |
| 鼠标左键 | 标记发现（寻找者专用） | 寻找者 |

> **原则：** 不引入 E/Q/R/F 等功能键。3个手指放在WASD上，小指够Ctrl，拇指够Space。寻找者唯一额外操作是鼠标左键。

### 核心规则：30秒位置暴露机制

```
倒计时：每30秒触发一次全局高亮
效果：所有躲藏者瞬间显示轮廓/光柱（持续约1秒）
目的：防止游戏陷入僵局，确保寻找者有追踪线索
```

| 参数 | 设定 |
|------|------|
| 触发间隔 | 每30秒 |
| 高亮持续时间 | 0.5 - 1.0 秒 |
| 高亮方式 | 角色位置出现瞬间亮色轮廓/光柱 |
| 可见性 | 仅寻找者可见 |
| 首次触发 | 搜索阶段开始后第30秒 |

> **设计意图：** 这是躲猫猫的"心跳"——每隔30秒给寻找者提供一次线索，同时给躲藏者制造"暴露窗口"的紧张节奏。躲藏者需要预判高亮时刻，考虑是否提前转移位置。

### 视角规则（全第一人称）

| 规则 | 说明 |
|------|------|
| 所有玩家视角 | **100% 第一人称**，无第三人称 |
| 躲藏者视角 | 第一人称躲藏，依赖听觉和有限视野判断寻找者位置 |
| 寻找者视角 | 第一人称搜索（FPS模板原生支持） |
| ❌ 偷看/探头 | **不实现** |
| ❌ 第三人称 | **不实现** |

> **设计意图：** 第一人称增强了躲藏者的沉浸感和紧张感——你只能听到脚步声，不知道寻找者在哪里，只能通过声音和余光判断。这比第三人称的"上帝视角躲藏"更具心理压力。

### 被抓捕后：观战模式

| 功能 | 说明 |
|------|------|
| **自由切换视角** | 在场上仍存活的玩家之间自由切换第一人称视角 |
| **上帝视角** | 或选择自由飞行摄像机俯瞰全局 |
| **切换方式** | 鼠标滚轮或数字键切换观战目标 |
| **观战者列表** | 屏幕显示当前存活玩家列表，可点击切换 |

> **设计意图：** 被淘汰≠退出游戏。观战者可以看到全局博弈，等待下一轮。

### 结算面板 — 计分系统

#### 躲藏者得分

```
躲藏者最终得分 = 存活基础分 + (存活时间 × 距离系数)

- 存活基础分：未被抓到 = +100分，被抓住 = 0分
- 存活时间分：每存活1秒 = +2分（最大120秒 = +240分）
- 距离系数：以整局中离寻找者最近距离的50%分位值为准
  - 极近（<2米）：系数 ×2.0
  - 近距（2-5米）：系数 ×1.5
  - 中距（5-10米）：系数 ×1.0
  - 远距（>10米）：系数 ×0.5
```

| 得分项 | 说明 |
|--------|------|
| 存活基础分 | 未被抓捕 +100，被抓 0 |
| 存活时间分 | 每秒 +2 分 |
| 距离加成系数 | 离寻找者越近躲藏，系数越高（高风险高回报） |

#### 寻找者得分

```
寻找者最终得分 = 抓捕基础分累计 + 衰减奖励 + 全捕获奖励

- 每次抓捕基础分：+150分/人
- 衰减奖励：第1个被捕者额外 +100，第2个 +70，第3个 +50，第4个 +35，第5个 +25，第6个 +15
  （奖励递减，早期抓捕价值更高）
- 全捕获奖励：抓到全部6个躲藏者 = +200额外奖励
- 时间奖励：剩余时间每秒 +1分（越快抓完分越高）
```

| 得分项 | 说明 |
|--------|------|
| 抓捕基础分 | 每抓到1人 +150 |
| 衰减奖励 | 越早抓到的奖励越高（第1个+100 → 第6个+15） |
| 全捕获奖励 | 6人全抓到 +200 |
| 时间奖励 | 剩余秒数 × 1 |

## 派对模式 — 四专家独立评审

### 🎲 Samus Shepard（游戏设计师）— 关卡设计与平衡性

**地图设计：**
- "三明治结构"：安全区/危险区/死亡区梯度分布
- 三层垂直空间，每层至少两条互不交叉楼梯
- 2000-3000平米，15-20个躲藏点，5-8个可互动物体
- 声景设计 > 视觉设计：不同地面材质提供不同脚步声

**平衡性：**
- 6v1 + 全程第一人称 = 天平严重倾向hider
- 30秒高亮1秒太短 → 建议1.5秒高亮+2秒轮廓残留
- 预警"蟑螂策略"（抱团）和"影子策略"（尾随seeker身后）
- 递减奖励改进：100→85→75→65→55→50，阶梯奖励替代全捕获奖励
- 近距加成是天才设计——需实时"冷/温/热"距离指示器

### 🏛️ Cloud Dragonborn（游戏架构师）— 网络架构与技术评估

**网络方案：** Unity Netcode for GameObjects (NGO)
- Unity 6.3 LTS第一方方案，出厂即融合
- Unity Transport + UDP直连足够LAN使用
- 不需要Mirror/FishNet的高级特性

**四大技术风险：**
1. Gamemode状态机（最核心）— Server侧NetworkVariable<GameState>
2. 高亮系统穿墙可见性 — MVP先用Minimap红点+方向指示器
3. 观战系统 — MVP最小方案：俯视地图
4. 得分同步 — 全部Server计算，禁止客户端计算

**最简架构六原则：**
1. Host即Server 2. Server权威Client顺从 3. NetworkVariable > RPC
4. 一个脚本一个职责（≤150行） 5. 不做Lobby（IP直连） 6. 先做俯视图高亮

**核心：Gamemode状态机是承重墙，其余皆装饰。**

### 🎨 Sally（UX设计师）— 新玩家体验与学习曲线

**首次引导：** 30秒单人引导局（安全沙盒），图形非文字提示，AI"差点发现"=最佳教学

**设计即教学：** 轮廓光引导躲藏点，心跳声替代30秒数字倒计时，角色设计暗示身份

**第一人称对10岁玩家的障碍：** 提供简洁雷达小地图、声音可视化、可开关第三人称辅助选项

**计分系统简化：** 对局中不显示分数，结算用趣味化呈现

**观众体验：** "幽灵帮凶"互动、信息差快感、快速切换高戏剧性视角、下一局预告

### 🎮 Indie（独游开发者）— MVP范围与开发路径

**MVP大幅裁剪：** 砍掉3回合制、计分系统、30秒高亮、观众自由飞行

**Phase构建顺序：**
- Phase 0 (1-2天): 两人同场景移动（Hello World）
- Phase 1 (3-5天): 1v1 + Tag + 倒计时 → 能玩了！
- Phase 2 (2-3天): 1vN + Lobby + 基础UI
- Phase 3 (持续): 正式地图 + 打磨 + 音效

**C#学习路线（按需）：**
1. MonoBehaviour生命周期 2. [SerializeField]+Inspector 3. NGO基础
4. Raycast 5. delegate/event 6. Coroutine

**五大杀手：** 联机调试挫败感、地图设计替代写代码、完美主义+教程地狱、没人联机测试、过度关注画面

> 一周后目标：两人灰盒场景 → 追人 → Tag → 120秒倒计时 → 结果。到达此点已超过90%烂尾项目。
